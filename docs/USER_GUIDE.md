# Token Manager 用户使用手册

## 认识三个数据来源

Token Manager 会在界面中区分：

- **本地日志**：Codex 本机状态库和日志；
- **本地代理**：从真实 API 响应的 `usage` 字段提取 Token；
- **官方余额**：厂商明确提供并已验证的余额接口。

三者不能混为一谈。没有官方账单权限的平台会显示限制说明，而不是虚构余额。

## 添加 API 账户

1. 打开“账户与模型”。
2. 找到平台，点击“官网”创建或管理 API Key。
3. 回到软件点击“添加账户”。
4. 填写便于识别的账户名称、官方 Base URL 和 API Key。
5. 点击“加密保存账户”。

同一平台可添加多个账户。账户关闭应用后仍会保留；可单独删除，也可批量清空。

## 开启 API 实时监控

1. 点击主界面右上角“一键开启 API 实时监控”。
2. 软件从 `18765` 开始为每个账户分配本机端口。
3. 复制对应账户的本机地址，例如：

```text
http://127.0.0.1:18765/v1
```

4. 在 SDK、编辑器或调用工具中，把原厂 Base URL 改成该地址。
5. API Key 可继续由 Token Manager 托管；代理会向原厂 HTTPS 地址转发。

代理只监听回环地址，不接受局域网连接。不要把 `127.0.0.1` 地址配置到另一台电脑或云服务器。

## 查看全部模型

“全部模型”页面提供：

- 今日全平台 Token、API 请求、活跃模型和成本；
- 7/30 天 Token 堆叠柱状图；
- 每个 AI 固定颜色和悬停明细；
- 模型用量排行、月度花费占比和 30 天消费趋势；
- API 代理和 CC Switch 连接状态。

Token 总量按输入与输出相加。缓存命中 Token 属于输入 Token 的一部分，不会重复相加。

## 添加单模型仪表盘

1. 先添加平台账户。
2. 打开仪表盘，点击“添加仪表盘”。
3. 从账户实际模型列表或内置目录中选择模型。
4. 新仪表盘只显示该平台、账户和模型的数据。

单模型页包含 Token、请求次数、缓存命中、成本、余额状态和四类近 7 天图表。

## Codex 专属页面

Codex 页面无需 API Key。首次使用前，在设置页填写个人 5 小时和 7 天 Token 预算。软件会依据本机日志显示预算剩余比例、滚动窗口和任务换算。

这些数字不是 ChatGPT 官方会员余额。官方客户端显示与本地估算冲突时，以官方客户端为准。

## 悬浮窗

- 在设置页点击“一键启用悬浮窗”；
- 拖动顶部空白区域移动窗口；
- 拖动右下角调整尺寸；
- 使用顶部下拉框切换模型；
- 在设置页勾选模块、拖动排序并选择两个迷你模块；
- 所有变更实时生效并自动保存。

## 导出账单

报告中心可以导出 UTF-8 CSV，字段包括时间、平台、模型、输入 Token、输出 Token、缓存 Token、成本和任务类型。Excel 可以直接打开该文件。

## 更换电脑

1. 在旧电脑设置页输入至少 8 位迁移密码。
2. 导出 `.tmbak` 文件。
3. 在新电脑安装 Token Manager。
4. 输入同一密码并导入迁移包。
5. 导入后的 API Key 会重新使用新电脑当前用户的 DPAPI 加密。

请把迁移包视为敏感文件，不要上传到 Issue 或公开网盘。
