# 新增模型适配器规范

每个供应商在 Rust 中实现 `ProviderAdapter`：

- `capability()` 返回官方账单、代理计量、账单导入能力以及面向用户的限制说明。
- `normalize()` 将供应商响应转换为 `UsageEvent`；必须产出平台、模型、ISO 时间、输入/输出/缓存 token、人民币成本和任务类型。
- 官方账单请求必须仅在用户主动同步时发起，使用 DPAPI 解密后的临时内存密钥；禁止写日志、禁止遥测、禁止把密钥传给第三方。
- 价格记录必须保存来源 URL、版本、生效时间、缓存时间、币种和缓存折扣规则；缺失字段时成本应显示“待计算”，而非猜测。
- 导入器必须对 CSV/XLSX 字段做严格校验，并保留原始账单行号以便用户追溯。

新增适配器的验收测试至少包含：成功响应、无余额权限、限流、网络断开、未知模型、缓存 token、币种换算和重复导入。界面卡片必须显式显示数据来源，不能把代理统计或推测值显示为官方用量。
